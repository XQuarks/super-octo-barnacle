# 世界观生成指南

本指南用于辅助 AI 为「以太叙事」生成可玩的、结构化的世界观。

---

## 1. 输入

用户创建世界时提供：

- **世界名称**：如「星辉魔法学院」「红楼梦·大观园」。
- **类型**：`ip`（已有作品）或 `original`（原创）。
- **世界观描述**：力量体系、社会结构、势力、地点、规则、时代背景。
- **主角设定**（可选）：出身、性格、初始目标。
- **文风选择**：模仿源文件 / 自定义 / 通用叙事。
- **剧情自由度**：1（严格遵循原著）到 5（完全自由）。
- **源文件上传**（可选）：EPUB / TXT / DOCX，从原文提取设定。

---

## 2. AI 输出

### 2.1 schema（世界属性模板）

定义角色在 UI 中的展示方式：

```json
{
  "progression_label": "境界",
  "progression_path_label": "修行路线",
  "has_skills": true,
  "skill_label": "功法/技艺",
  "attribute_labels": {
    "courage": "胆识",
    "perception": "洞察",
    "patience": "耐心",
    "luck": "气运",
    "will": "心志"
  },
  "time_periods": {
    "morning": "早晨",
    "forenoon": "上午",
    "afternoon": "下午",
    "evening": "傍晚",
    "night": "夜晚"
  },
  "game_over_conditions": ["is_alive === false"]
}
```

| 字段 | 设计原则 |
|------|---------|
| `progression_label` | 贴合世界观：修仙→境界，魔法学院→年级 |
| `has_skills` | 无技能系统设为 false |
| `attribute_labels` | 五维属性中文名，可按世界观调整 |
| `time_periods` | 一天内时段，可增删并自定义名称 |

### 2.2 initial_state（玩家初始状态）

包含 `name`、`background`、`personality`、`attributes`、`progression`、`relationships`、`skills`、`inventory`、`location`、`date`、`goals`、`npc_activity` 等字段。

**关键规则**：
- `attributes` / `relationships` / `skills` 使用**文字描述**而非数字。
- `goals` 含阶段目标和最终目标，可设 deadline。
- `npc_activity` 记录每位 NPC 的初始位置和状态。
- `is_alive` 必须存在。

### 2.3 lore_kb（知识库）

必须覆盖以下 category：

| category | 最少条目 | 要求 |
|----------|---------|------|
| 规则 | 1 | 世界运行的硬规则 |
| 地点 | 1 | 初始地点及周边 |
| 人物 | 1 | NPC 性格、态度、**日常行程** |
| 冲突 | 2 | 核心矛盾与张力（如门派之争、家族恩怨） |
| 事件 | 2 | 可触发事件，含触发条件（时间+地点+前置） |
| 势力/物品 | 按需 | |

每条片段 100-300 字，需 `keywords`（3-8 个）。

### 2.4 system_prompt

见 `03_System_Prompt工程.md`。

### 2.5 opening_narrative（开场白）

1-3 段沉浸式叙事，用于玩家首次进入世界。

### 2.6 initial_choices（初始选项）

2-4 个初始行动选项，每项含 `text` 和 `hint`，符合世界观和角色设定。

---

## 3. 检查清单

- [ ] 力量体系自洽，不可跳跃
- [ ] 是否有不可篡改的核心剧情/角色命运
- [ ] NPC 有初始态度、行事理由和日常行程
- [ ] 死亡条件合理
- [ ] 物品/技能/地点属于该世界
- [ ] 初始目标明确可达成
- [ ] 至少 2 条冲突类知识片段
- [ ] 至少 2 条事件类知识片段
- [ ] NPC 关系与事件触发条件不矛盾
