# 状态与数值设计

角色状态系统使用**文字描述优先**的设计，避免数字膨胀和机械感。

---

## 1. 五维属性

| 属性 | 默认中文名 | 含义 |
|------|-----------|------|
| courage | 胆识 | 面对危险/冲突的反应能力 |
| perception | 洞察 | 观察力、直觉、察言观色 |
| patience | 专注 | 耐心、持久力、自控力 |
| luck | 幸运 | 偶然事件、机遇、巧合 |
| will | 意志 | 坚韧、抗压、不轻易放弃 |

属性使用**文字描述**而非整数数值。AI 每次叙事中通过文字体现属性水平的变化：
- "你面对修士威压已不再颤抖"（胆识提升）
- "你能从嘈杂中分辨出有用的声响"（洞察提升）

属性更新规则：只在发生**质变**时更新描述，不每次微调。

---

## 2. 关系系统

NPC 关系同样使用**文字描述**，记录玩家与每位 NPC 的当前关系状态：

```
"贾母": "老太太疼你，说你眉眼间有几分她年轻时的样子"
"林黛玉": "你们一见如故，她常说你是这府里唯一懂她的人"
```

关系描述通过 user message 每轮完整注入，作为 AI 保持角色行为一致性的依据。

---

## 3. NPC 状态账本（npc_states）

NPC 的位置与"人味"由结构化 `npc_states` 账本统一管理，作为权威来源（早期仅自由文本的 `npc_activity` 已升级为结构化字段）：

```json
"林黛玉": {
  "attitude": 12,          // -100..100，量化好感（跨阈值自动触发关系质变）
  "mood": "警惕但好奇",
  "location": "潇湘馆",    // 权威位置，AI 移动 NPC 必须同步更新
  "schedule": { "morning":"潇湘馆", "night":"沁芳亭" },
  "secrets": ["她隐瞒了……"],
  "speech_style": "说话慢条斯理，爱用比喻",
  "catchphrase": "「这世道啊……」",
  "card": { "appearance":"", "personality":"", "desc":"", "notes":"" }  // 玩家手动设定的人物卡
}
```

- **位置账本**：进入某地点时，引擎列出 `npc_states` 中 `location===该地点` 的 NPC，提示 AI"此刻应在此处"，避免 NPC 无理由瞬移（沉浸感一致性）。
- **好感量化**：`attitude` 由 AI 在 `state_changes.npc_states` 中微调（如 ±5），引擎合并而非替换；跨阈值生成"关系质变"事件。
- **声音一致性**：`speech_style`/`catchphrase` 注入 system prompt，要求 AI 严格保持 NPC 说话腔调。
- **人物卡**：玩家可在状态面板「关系」页签为任意遇到的 NPC 设定外貌 / 性格 / 描述 / 备注，存入 `card` 字段。

## 4. 技能系统

技能使用**文字描述**，按世界观自定义：
- 修仙世界：剑术/打坐/炼药
- 魔法学院：基础元素操控/魔法理论/炼金术
- 红楼梦：诗词/琴艺/书画/女红

---

## 5. 物品系统

```json
{
  "item_id": "jade_pendant",
  "name": "羊脂玉佩",
  "count": 1
}
```

支持 `add`/`remove` 操作，AI 可决定何时给予或消耗物品。

---

## 6. 目标系统

| 类型 | 特征 |
|------|------|
| 可见目标 | 明确告知玩家，含 deadline |
| 隐藏目标 | NPC 暗示或线索触发后变为可见 |
| 目标链 | 完成 → 生成新目标，形成因果链 |
| 失败分支 | deadline 过后标记 failed，叙事体现后果 |

---

## 7. 状态持久化

所有状态通过 `localStorage` 持久化，每次 AI 响应后保存。存档同时保存 `chatSummary`（对话历史摘要）用于跨轮次记忆。
