# System Prompt 工程

System Prompt 是控制 AI 叙事行为的核心。当前使用固定前缀 + 动态后缀结构，利用 DeepSeek 磁盘缓存大幅降低响应成本。

---

## 1. 结构概览

```
# 角色设定
# 世界观硬约束
# 叙事风格
# 叙事节奏控制（三模式）
# 选项设计要求（五条规则）
# NPC 自主行动
# 事件触发条件（复合条件）
# 数值对叙事的约束
# 目标系统（链/隐藏/失败分支）
# 世界属性模板
# 你的职责
# 输出格式（JSON Schema）
# 状态变更规则
# 禁止事项
{RULES_MODE_SECTION}      # 双模式规则段：纯叙事 NARRATIVE_RULES_BLOCK / 跑团 TRPG_RULES_BLOCK（按 ruleset_type 注入）
{NARRATIVE_QUALITY_SECTION} # 叙事质感段：五感/节奏/留白指引，可按 IP 风格切换
---
# 动态内容（实际走 user message，见下文）
# 当前游戏状态
# 相关知识片段
# 玩家行为记录（关键事实）
# 玩家输入
```

> **双模式（叙事 vs 跑团）**：system prompt 在构建时按 `currentWorld.ruleset_type` 注入不同的规则段——`narrative`（纯叙事，无骰子、无数值成长，改用名号/声望/传说度反馈）或 `dnd`/其他（TRPG，含 D20 规则系统）。同一世界内该段恒定，确保前缀缓存命中。

> **模板静态化**：早期模板含 `<!-- DYNAMIC -->` 及 `{GAME_STATE_JSON}`/`{RETRIEVED_LORE_SNIPPETS}` 等占位符，但构建时只取 DYNAMIC 之前部分、动态内容全部走 user message，占位符实际永不被替换（死代码）。现已从模板中移除这些死代码占位符，仅保留 `{RULES_MODE_SECTION}` 与 `{NARRATIVE_QUALITY_SECTION}` 两个真实占位符。

---

## 2. 缓存优化策略

### 2.1 前缀缓存锚定

- system prompt 构建后**硬化缓存**，同一世界内永远返回相同字符串
- 聊天历史使用**锚定模式**：前 2 轮作为稳定锚点（永不删除）+ 最近 2 轮灵活追加
- 动态内容（关键事实、NPC 关系、游戏状态、玩家输入）**全部放在 user 消息**
- 不在 system prompt 中嵌入时间戳、随机 ID 或任何每轮变化的内容

### 2.2 Token 优化

- 聊天历史压缩：完整保留最近 4 轮，更早轮次用 1-2 句中文摘要
- 游戏状态精简：仅传变化字段，去掉了 personality/background/name/age 等静态字段
- assistant 消息精简：仅保留 narrative + state_changes，去掉 choices/key_facts

---

## 3. 叙事节奏控制

根据场景自动切换三种模式，确保阅读体验有张有弛：

| 模式 | 触发条件 | 输出特征 |
|------|---------|---------|
| **展开模式** | 场景切换、重要 NPC 登场、关键事件 | 2-3 段长叙事，重描写、铺氛围 |
| **推进模式** | 日常交互、对话交谈、小型行动 | 1-2 段精炼叙事，节奏紧凑 |
| **加速模式** | 重复动作、时间跳跃过渡 | 1 段概括，快速推进 |

进入新场景或重要事件时立即切回展开模式。

---

## 4. 选项设计

每轮 2-4 个选项必须满足：

1. 至少一个带明确代价或风险
2. 至少一个引导玩家表达态度或情感
3. 至少一个与当前场景/角色独特性相关
4. 禁止空泛选项（"继续前进""再试一次"）
5. 暗示可能结果但不剧透

---

## 5. NPC 自主行动

- NPC 有自己的行程和状态，不总是在原地等玩家
- 玩家进入地点时，描述当前在此处的 NPC 及其状态
- NPC 之间可能互动，玩家可能撞见
- 每次叙事可含 1-2 句 NPC 独立行为描写
- 状态通过 `npc_activity` 字段持久化传递

---

## 6. 事件触发

事件不再仅依赖关键词，而是基于复合条件：

- **时间条件**：特定时段或日期
- **地点条件**：玩家到达特定位置
- **数值条件**：属性或关系达到阈值
- **前置事件**：已完成事件作为前提

---

## 7. 数值约束

属性与关系必须真实影响行为：
- 胆识低 → 叙事中体现犹豫退缩
- 灵慧高 → 注意细节，听出言外之意
- 关系冷淡 → NPC 敷衍回避
- 关系亲密 → NPC 主动透露信息

---

## 8. 目标系统

- **目标链**：完成一个目标后生成相关新目标
- **隐藏目标**：NPC 暗示的任务，触发后加入可见
- **失败分支**：deadline 过后标记 failed，叙事体现后果

---

## 9. 时间推进

基于行动实际耗时，非机械每轮推进：

| 行动类型 | 时间推进 |
|---------|---------|
| 短对话、观察、简单移动 | 不推进 |
| 长篇交谈、跨院落拜访、小型宴席 | 推进 1 时段 |
| 出府远行、大型宴会、重大事件 | 推进 2-3 时段 |

核心原则：优先保持时间不变，只在场景自然切换时推进。

---

## 10. 剧情自由度控制

| 级别 | 规则 |
|------|------|
| 1 | 严格遵循原著，NPC 不可偏离命运线 |
| 2 | 原著为主，日常可发散，NPC 次要行动自主 |
| 3 | 框架内创新，NPC 依性格自主行动 |
| 4 | 世界观为框架，NPC 完全自主 |
| 5 | 仅基本设定为框架，全部自由创造 |

---

## 11. 输出格式

严格 JSON，包含 `narrative`、`choices`、`state_changes`（含 `npc_activity`）、`key_facts` 等字段。
