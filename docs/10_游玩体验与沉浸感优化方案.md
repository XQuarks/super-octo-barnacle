# 10. 游玩体验 / 游玩内容 / 沉浸感 优化方案

> 目标：在现有架构（RAG 检索、`behavior_records` 关键事实、`npc_states` 位置账本、双模式、`goals`+`deadline`、`status_effects`、`progression`）之上，提升「世界是否活」、「玩家是否有抓手」、「是否身临其境」三件事。
> 每项标注：问题 → 方案 → 落点（文件/字段/机制）→ 优先级。字段设计尽量复用既有结构，降低改动面。

---

## 一、游玩内容：让世界"活"起来、有得玩

### E1 世界脉搏 World Pulse（动态事件 / 生态）— P0
- **问题**：世界是 AI 一次性生成的静态 `lore_kb` 片段。玩家行动后世界不会"自行发生事"——没有天气、传闻、势力动向，沉浸感断崖。
- **方案**：每推进 N 个时段（建议 2 时段），引擎基于 `gameState` + `behavior_records` 生成一条「世界动态」，写入 `world_events` 并经 RAG 注入下一轮。约束 prompt：「不推进玩家主线、仅作氛围/可选线索，避免抢戏」。
- **落点**：
  - `app-game.js` 的 `applyTimeChange` 增加时段计数；跨阈值调 `generateWorldPulse()`。
  - `app-ai.js` 新增 `generateWorldPulse()`，复用 LLM 调用；结果 `addBehaviorRecords`-式写入。
  - 检索路径（`retrieve`）追加 `world.current_world_events` 来源。
- **字段**：
  ```json
  "current_world_events": [
    { "id":"we_001", "day":3, "period":"afternoon",
      "type":"rumor|env|faction|weather",
      "text":"镇上流传：矿洞深处又亮起幽蓝的光。",
      "impact_tag":"mine_mystery" }
  ]
  ```
- **注意**：纯叙事与跑团都应触发，但跑团世界可由 `ruleset` 控制频率。

### E2 NPC 深度卡（好感度量化 + 日程 + 秘密 + 说话腔调）— P0
- **问题**：`relationships` 只是自由文本（"萍水相逢，对方看你的眼神里带着几分打量"），无量化好感、无日程、NPC 没"人味"；**你（黎总）明确对好感度有兴趣**。
- **方案**：把已有 `npc_states`（位置账本）扩展为完整 NPC 卡。好感 `attitude` 量化后，AI 可在 `state_changes.npc_states` 里微调（如 +/-5），跨阈值自动触发「关系质变」事件并记 `behavior_records`。
- **落点**：
  - `initial_state.json` / 世界生成 prompt 增加 NPC 模板。
  - `app-ai.js` 在 per-turn 注入 NPC 卡（含 `catchphrase`/`speech_style` 强化"声音一致性"）。
  - `app-game.js` 的 `applyStateChanges` 已处理 `npc_states`，补 attitude 合并。
- **字段**：
  ```json
  "npc_states": {
    "guide_npc": {
      "location": "入口小镇·客栈",
      "attitude": 12,            // -100..100，量化好感
      "mood": "警惕但好奇",
      "schedule": { "morning":"客栈", "night":"镇外哨塔" },  // 已有位置账本扩展
      "secrets": ["他隐瞒了自己曾是……"],
      "speech_style": "说话慢条斯理，爱用比喻",
      "catchphrase": "「这世道啊……」"
    }
  }
  ```

### E3 目标系统升级（分层 + 进度 + 世界压力）— P1
- **问题**：`goals` 无进度、无主线/支线分层；`deadline` 到了（B1 已强制 `failed`）但世界"毫无反应"，玩家没痛感。
- **方案**：
  - goal 增加 `progress`(0-100)、`tier`('main'|'side'|'personal')、`sub_goals[]`。
  - B1 `checkGoalDeadlines` 标记 `failed` 时，额外生成「错过后果」叙事（如错过相亲→家族不满，自动记 `behavior_records`）。
  - 临近 `deadline`（剩 1 时段）注入「紧迫感」提示给 AI，引导世界施加压力。
- **落点**：`app-game.js` 的 `checkGoalDeadlines`、`app-ai.js` 的 `buildSystemPrompt`/per-turn 注入。

### E4 叙事层成长 / 声望系统 — P1（叙事模式关键补短板）
- **问题**：叙事模式无数值成长，`progression`（`path/rank/progress`）字段被创建但从未驱动，玩家缺乏「我变强了 / 世界因我而变」的反馈。
- **方案**：叙事模式用「名号 / 声望 / 传说度」替代属性。关键行为累积 `reputation`，跨阈值解锁「头衔」写入 `behavior_records` 并触发世界反应（路人认得你、NPC 态度偏移）。
- **落点**：`app-game.js` 维护 `reputation`；`isNarrativeMode()` 下用该字段渲染成长反馈，`buildCompactGameState` 注入。

## 二、游玩体验：节奏、反馈、可见性

### E5 抉择日志 / 命运簿（选择后果可视化）— P1
- **问题**：选项点了就过，玩家看不到「我的决定留下了什么」，蝴蝶效应感弱。只有后台 `behavior_records`。
- **方案**：结构化记录关键选择（带 `tag` 与后果倾向），UI 侧边栏可回看，强化「每一步都算数」。
- **落点**：`app-ai.js` 在生成 `options` 时要求 AI 返回 `choice_consequence` 摘要；`app-ui.js` 列表渲染；数据可归并进 `behavior_records` 或独立 `milestones`。

### E6 多视图侧边栏（状态 / 关系 / 目标 / 日志 / 地图草图）— P0（体验质变）
- **问题**：纯聊天式，玩家频繁追问「我在哪 / 他是谁 / 我接下来干啥」，打断沉浸。
- **方案**：右侧 Tab 面板，信息分层，**不污染叙事流**。数据全部来自 `gameState` 既有字段（`npc_states`/`relationships`/`goals`/`inventory`/`current_world_events`）。
- **落点**：`app-ui.js` 渲染；纯前端，不影响核心循环。可先上「关系 + 目标 + 日志」三 Tab。

### E7 危机 / 张力机制（非战斗）— P2
- **问题**：除战斗外无张力，叙事易「温吞」。
- **方案**：叙事层「悬念值」——被某势力盯上 / 某秘密将暴露 / 时间炸弹，随时间或行动升级，触发专属提示与专属选项，制造压迫感。

### E8 创建角色叙事锚点 — P1
- **问题**：创建时只填属性，缺「我是谁」的叙事钩子（你重视「玩家可自定义背景」）。
- **方案**：创建引导填「一个执念 / 一个秘密 / 一个禁忌」，写入 `behavior_records` 锚点；NPC 后续可引用（"你说过你在找……"），强化代入。
- **落点**：`app-ui.js` 创建流程 + 世界生成 prompt。

## 三、沉浸感：描写、一致性、感官

### E9 沉浸描写指引（五感 / 节奏 / 留白）— P0（低成本高回报）
- **问题**：`system_prompt_template.md` 目前只管「叙事一致性自检」，没引导描写风格，AI 输出易写成「信息报告体」而非「小说体」。
- **方案**：模板增加「叙事质感」段，可按 IP 风格切换——武侠重意境留白、克苏鲁重氛围压迫、校园重日常幽默、科幻重冷峻精确。
- **落点**：`system_prompt_template.md` 加 `{NARRATIVE_QUALITY_SECTION}` 占位符；`buildSystemPrompt` 按 `world.style` / `ruleset` 注入对应段。
- **示例指令**：
  ```
  描写以"展示而非告知"为主；优先用动作、环境、对话传递信息。
  每 3-5 段至少一处具体感官细节（气味/温度/光线/触感）。
  克制解释性旁白，让读者自己体会。
  ```

### E10 多感官 / 环境锚点注入 — P2
- **问题**：场景缺「此时此地」的感官锚，玩家难身临其境。
- **方案**：每时段基于 `current_location` + `period` + `weather` 生成 1-2 句环境锚点，注入 user prompt 顶部，作为 AI 描写引子（而非替 AI 写满）。
- **落点**：`app-ai.js` per-turn 注入；可复用 `generateWorldPulse` 的环境分支。

### E11 NPC "声音"一致性强化 — P1
- **问题**：NPC 性格有自由文本，但「说话腔调 / 口头禅」易漂移（E2 已补 `catchphrase`/`speech_style`）。
- **方案**：将 NPC 卡 `speech_style`/`catchphrase` 注入 system prompt 的「叙事一致性」段，要求 AI 严格保持。

### E12 跨周目传承（传奇摘要）— P2
- **问题**：每次世界独立，无积累感。
- **方案**：通关 / 结束时生成「传说摘要」，可导出或带入新世界作为彩蛋——新世界 `lore_kb` 注入一段「久远传说里记载过……」，让老玩家会心一笑。

## 落地路线图

| 批次 | 项 | 理由 |
|------|----|------|
| **第一批（低成本高回报，纯提示/轻量前端）** | E9、E6、E2（好感度基础）、E1 | E9 仅改 prompt；E6 纯前端；E2 复用 `npc_states`；E1 复用 RAG。这四项直接拉升"世界活 + 身临其境 + 有抓手"。 |
| **第二批** | E3、E4、E5、E8 | 目标/声望/抉择日志/创建锚点，强化"成长感与因果感"。 |
| **第三批** | E7、E10、E11、E12 | 张力/感官/声音/传承，做满沉浸上限。 |

**建议起点**：先做 E9 + E6 + E2，三项就能让"纯叙事"和"跑团"两种模式都明显更有戏——且不碰核心循环、风险低、可快速在浏览器实测。确认方向后我再把第二批、第三批逐批落地。

## 四、实测反馈落地（UX 优化）

基于浏览器实测发现的体验问题，已逐项修复：

### UX-1 每次都能设定主角 + 存档显示主角 — 已修复
- **问题**：仅第一次进入世界能设主角，二次开始冒险无法修改；存档列表看不到主角设定。
- **方案**：游戏头部新增「编辑主角」按钮 → `heroEditModal` 随时设定/修改主角身份与背景；`saveHeroEdit` 同时写 `currentWorld.hero` 与存档对象 `s.hero`，**立即**反映到主界面存档列表（不再等下一回合自动存档）。
- **落点**：`app-ui.js` `openHeroEdit`/`saveHeroEdit`；存档列表 `.item-hero` 展示「主角：…」。世界详情弹窗 `heroDescEdit` 作为进世界前入口，两处写同一字段互不冲突。

### UX-2 删除框被标签框遮挡 — 已修复
- **问题**：世界卡片的原创/IP 标签框（`position:absolute`）压住了下方的删除按钮。
- **方案**：`.wc-actions` 改为普通流（`margin-top:10px`），HTML 中标签框 `.wc-tags` 在删除按钮之前，删除按钮稳稳落在标签下方。
- **落点**：`index.html` 状态面板/世界卡片样式。

### UX-3 每个 NPC 可设人物卡 — 已修复
- **问题**：NPC 仅有 AI 生成的关系描述，玩家无法补充自己视角的外貌/性格/备注。
- **方案**：新增「人物卡」弹窗（外貌 / 性格 / 描述 / 备注），存入 `npc_states[name].card`；状态面板「关系」页签每个 NPC 卡片下方提供「＋ 设定人物卡 / 编辑人物卡」按钮。
- **落点**：`app-ui.js` `editNpcCard`/`saveNpcCard` + `index.html` `npcCardModal`。

### UX-4 状态面板信息过载 — 已修复（重构为 5 分组）
- **问题**：原 10 个页签（profile/background/state/relations/items/skills/goals/npc/world/log）信息太碎、按钮太多。
- **方案**：合并为 **5 个分组页签**：
  - **角色**（profile + background + state + skills 并入）
  - **关系**（relations + npc 人物卡并入）
  - **收集**（items）
  - **目标**（goals）
  - **世界动态**（world + 抉择日志 log 并入）
  - 属性详情、战斗数值等次要信息用 `<details>` 折叠收起，并补 `.status-details` 折叠样式。
- **落点**：`app-ui.js` `renderStatusTabs` / `renderStatusPanel` 全量重构。
