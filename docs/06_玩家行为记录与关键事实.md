# 玩家行为记录与关键事实

行为记录确保长对话中 AI 不会遗忘关键剧情。

---

## 1. 工作流程

```
AI 叙事输出 → 提取 key_facts → 加入 behavior_records
     ↓                                      ↓
  触发 applyStateChanges              retrieveBehaviorRecords
     ↓                                      ↓
  更新 gameState                     对比玩家输入 + 中文分词
                                           ↓
                                      匹配关键事实 → 注入 user message
```

---

## 2. 关键事实生成

AI 每轮输出 JSON 时附带 `key_facts` 数组：

```json
{
  "key_facts": [
    "玩家获得了草药",
    "玩家与老道长的关系有所提升"
  ]
}
```

前端自动过滤重复项（`behavior_records` 中已存在的跳过），上限保留 100 条。

---

## 3. 永久记忆注入

不同于 RAG 检索的按需召回，关键事实采用**无条件注入**策略：

- 每轮 user message 开头注入最近 **5 条**关键事实作为"已发生的关键事件"
- 标题强调"务必记住"，确保 AI 不遗忘
- 无论检索结果如何，关键事实始终注入

---

## 4. 行为记录检索

当玩家输入涉及过去事件时，通过 `retrieveBehaviorRecords()` 匹配相关行为记录：

- 使用 `Intl.Segmenter` 中文分词（与关键词检索一致）
- 按输入与行为记录文本的匹配度评分
- 得分 > 0 的记录加入融合排序

---

## 5. 对话历史摘要

为减少 API 请求的 token 消耗：

- chatHistory 只保留最近 4 轮完整对话（8 条消息）
- 每轮 AI 回复后自动生成 1-2 句中文摘要（`chatSummary`）
- 摘要取叙事文本前 2 个句子，含地点变更信息
- 摘要上限 20 条，防止无限增长

---

## 6. 玩家输入 → AI 输出全流程

```
玩家输入
  ↓
中文分词 → RAG 并行检索（关键词 + 向量 + 行为记录）
  ↓
融合排序 → Top 8 片段
  ↓
构建 user message:
  - 对话历史摘要
  - 最近 5 条关键事实（永久记忆）
  - NPC 关系描述
  - RAG 检索片段
  - 紧凑游戏状态
  - 玩家输入
  ↓
构建 messages: [system(缓存) + chatHistory(锚定) + user(动态)]
  ↓
发送 API → SSE 流式响应
  ↓
解析 JSON → 应用状态变更 → 生成摘要 → 归档关键事实 → 保存
```

---

## 7. 后端无状态设计

所有行为记录存入 `currentWorld.behavior_records`（localStorage 持久化），无需服务器。读档时完整恢复。ID 使用 `crypto.randomUUID()` 保证唯一性。
