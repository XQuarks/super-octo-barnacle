# 知识库设计

知识库是 RAG 检索的核心，约束 AI 不编造、不跨界。每条片段以 100-300 字为宜。

---

## 1. 结构

```json
{
  "ip": "世界名称",
  "snippets": [
    {
      "id": "x1",
      "category": "规则",
      "title": "修行境界",
      "content": "凡人→练气→筑基→金丹→元婴……境界不可跳跃，需按部就班修行。",
      "keywords": ["境界", "修行", "突破"],
      "embedding": [0.12, -0.05, ...]
    }
  ]
}
```

---

## 2. 分类体系

| category | 用途 | 最少条目 |
|----------|------|---------|
| 规则 | 世界运行的硬规则 | 1 |
| 地点 | 场景设定、外观、限制 | 1 |
| 人物 | NPC 性格、态度、**日常行程** | 1 |
| 冲突 | 核心矛盾与张力（必填） | 2 |
| 事件 | 可触发事件 + 触发条件 | 2 |
| 势力 | 组织与派系 | 按需 |
| 物品 | 重要道具 | 按需 |
| 世界观 | 整体氛围、时代背景 | 按需 |

**冲突与事件是新增的关键分类**：冲突告诉 AI 世界有什么矛盾可驱动剧情；事件提供具体的触发剧本（时间+地点+关系阈值）。

---

## 3. 关键词

每条片段 3-8 个关键词，用于关键词检索和粗排。覆盖：名称、别名、相关地点、人物、核心概念。

---

## 4. 向量嵌入

- 模型：`sentence-transformers/all-MiniLM-L6-v2`（384 维）
- 文本拼接格式：`[category] title\ncontent`
- 生成工具：`tools/generate_embeddings.py`
- 浏览器端推理：`@xenova/transformers` 支持

---

## 5. 检索策略（混合 RAG）

每次玩家输入触发：

1. **关键词检索**（中文分词 `Intl.Segmenter`）：快速召回匹配关键词的片段，加权评分
2. **向量语义检索**（Xenova/all-MiniLM-L6-v2）：余弦相似度排序
3. **行为记录检索**（中文分词）：匹配已发生的关键事实
4. **融合排序**：关键词分 + 向量分加权合并，取 Top 8

关键词检索和向量检索**并行执行**（`Promise.all`），减少等待时间。

---

## 6. 恒注入规则

无论检索结果如何，以下始终可用：

- 世界观核心规则（境界/等级/禁忌）
- 玩家当前地点相关信息
- 当前涉及 NPC 的性格态度
- 永久关键事实（最近 5 条，无条件注入 user message）
- NPC 关系描述（完整注入 user message）

---

## 7. 知识库生命周期

- **创建**：AI 根据世界观描述自动生成 12 条片段
- **上传**：EPUB/TXT/DOCX 源文件可提升精确度
- **维护**：长对话中关键事实自动归档到 `behavior_records`
- **容量**：关键事实保留最近 100 条

---

## 8. 维护建议

- 审查 AI 是否频繁编造知识库外内容 → 补入知识库
- 冲突类片段限 2-3 条，避免 AI 制造过多矛盾
- 事件片段应具体可触发，不超过 5-8 条
- NPC 行程要具体（时间+地点），不用泛化描述
