# 常驻记忆层（pinned_facts）设计文档

> 对应优化项：竞品对比报告中的「故事性 ⑧ 常量记忆层」
> 状态：已落地 + 真实 API 端到端验证通过（2026-07-07）

---

## 1. 问题根因

原系统只有一层"关键事件"记忆，实现为滑动窗口：

```js
// js/app-ai.js
function getRecentKeyFacts(count) {
    const records = currentWorld.behavior_records;
    return records.slice(-count).map(r => r.text); // 永远只取最后 N 条
}
```

- `behavior_records` 上限 100 条，强制注入的只有最后 5 条（`getRecentKeyFacts(5)`）。
- 长尾事实（更早的誓言/诅咒）只在**玩家主动提起**（RAG 关键词命中 ≥2）时才被召回。
- 后果：**一个玩家没再提起的沉默誓言/血咒，会随着游戏推进悄悄滑出强制注入区，被 AI 慢慢遗忘**——长线剧情漂移。这正是同类产品（如 AI Dungeon 2026 仍被"剧情遗忘"困扰）的最大痛点。

要点："玩家立誓"和"玩家买了把剑"被当成同优先级。**买剑忘了无所谓，立誓忘了故事就崩。** 缺的是一条"一旦成立就永远成立"的专用通道。

---

## 2. 四层记忆架构

| 层 | 内容 | 注入方式 | 现状 |
|---|---|---|---|
| World Bible | 世界观设定 | 生成时常驻（命中缓存） | 已有 |
| **Pinned Facts（常驻层）** | 誓言/诅咒/生死/阵营/不可逆世界改变 | **全量无条件注入，永不随窗口滑动褪色** | **本次新增** |
| Sliding Key Facts | 最近 5 条剧情 | 滑动窗口 | 已有 |
| RAG behavior_records | 长尾细节 | 命中 ≥2 才召回 | 已有 |

新增的 `pinned_facts` 独立于滑动窗口，是"永远在场"的事实层。

---

## 3. 数据结构

挂在 `currentWorld` 上（与 `behavior_records` 并列）：

```js
currentWorld.pinned_facts = [
  {
    id: "p3941113b",                                  // 唯一 id
    text: "玩家向森林女神立下誓言：此生绝不踏入北境禁地",
    pinnedAt: "2026-07-07T10:33:47.283Z",
    status: "active",                                // active | unpinned
    source: "ai"                                      // ai（AI 标注）| state（状态信号兜底）| manual（玩家手动）
  }
];
```

- **无 100 条上限**（与 `behavior_records` 不同）。长篇也就 10–30 条，全量注入 token 成本可忽略；真超过 40 条再做相似度筛选。
- 解除后保留为 `status: "unpinned"` 的历史记录，避免 AI 把已完结的誓言继续当枷锁。

---

## 4. 自动钉住双保险（核心）

设计原则：**玩家不需要手动添加记忆**（这是与 NovelAI 纯手写 Memory 的本质区别，也是本项目该有的架构优势——既然已有 AI 自动提取 `key_facts` 的能力，常驻层也应自动化）。

### 机制 A — AI 标注
`buildSystemPrompt` 的真实 system 指令里，要求 AI 在发生不可逆事实时写入 JSON 的 `pinned_facts` 字段：
- 立下的誓言/承诺、缔结的契约/pact
- 施加或身负的诅咒/祝福
- 角色死亡（含 NPC）、阵营归属
- 不可逆的世界改变（城破、纪元更替）、核心主线目标

落库处（`js/app-game.js` `processTurn`）把 `resp.pinned_facts` 送入 `addPinnedFacts("ai")`。

### 机制 B — 状态信号兜底（结构化硬保证）
`applyStateChanges` 识别语义上恒久的状态键，写入即自动镜像进 `pinned_facts`，**不依赖 AI 自觉**：
- `is_alive === false`（死亡）
- `cursed`（诅咒）
- `oath` / `oath_*`（誓言）
- `pact`（契约）
- `allegiance`（阵营）
- `deaths[]`（NPC 死亡）

---

## 5. 注入点

在 `buildTurnUserMessage` 的"永久记忆"块**之前**注入：

```text
# 恒定事实（无论如何不可违背、不可遗忘）

1. 玩家向森林女神立下誓言：此生绝不踏入北境禁地
```

全量、无条件、每轮都在。AI 必须始终遵守、不可遗忘或违背。

---

## 6. 解除机制

- AI 在叙事中体现誓言完成/诅咒解除时，停止继续写入该条，原条转为 `unpinned`。
- 玩家可在状态面板「记忆」标签手动解除一条，或补一条 AI 漏掉的。
- 保留 `unpinned` 历史，避免 AI 误把已结束的约束当枷锁。

---

## 7. 玩家角色

**可选的审阅者，不是录入者。**
- 正常游玩零操作，引擎自动钉住，玩家无感知。
- 状态面板「记忆」标签页：列出所有 active 恒定事实（带来源标记 + 解除按钮）、手动添加输入框、已解除历史区。
