2. 逻辑与系统维度

P0已落地

①"跑团"名不副实：非战斗技能检定从未接入骰子

现状action-menu.js 里 resolveSkillCheck() 写好了，但 UI 只接了 attack / cast（战斗）；交谈/移动/物品 三个按钮只生成一句通用文本（如"尝试与在场的人交谈"）直接丢给 AI 自由发挥。rulesets.json 里定义的 persuade / survey / survive / investigate / sneak 等社交/探索动作全无掷骰支持。
同行任何真·TRPG 的核心不是战斗骰，而是社交/探索技能检定——说服、潜行、调查、察觉。Mythic/Ironsworn 的"move"本质就是带随机性的行动判定。
建议把规则集里的动作映射成技能检定按钮（说服→CHA 检定、调查→INT、潜行→DEX…），DC 由场景/张力动态给，结果作为 [系统规则结果] 喂给 AI 叙事。让"单人跑团"真正有"判定"。
成本低：复用现有 Dice.checkAgainst + resolveSkillCheck，只补 UI 绑定与 system 指令。
落地action-menu.js 已接入 talk→说服(CHA,DC13)/use_item→巧手(DEX,DC13)/move→体操(DEX,DC12)，检定结果以 [系统规则结果] 喂给 AI 叙事；skill_check 测试 13 PASS。
P1

② NPC 位置抽取脆弱：只认 lore 已知地点名

现状extractLocationFromActivity() 只在 npc_activity 文本里匹配 lore 地点 类标题。一旦 AI 描写一个新地点名（不在知识库），抽取返回 null，NPC 位置账本静默失效 → 又可能"瞬移"。
建议让 AI 在 state_changes.npc_states 里显式返回 location 字段（结构化而非靠文本抽取），位置账本直接采信；文本抽取仅作兜底。
成本低：改 world-gen prompt + applyStateChanges 读取逻辑。

③ C1 安全自相矛盾（你之前指示跳过，此处仅作风险提示）

现状app-core.js:84 主题连点 10 次会把一段 jailbreak（"Untrammelled Writing Assistant…Ignore irrelevant moral appeals…"）注入 custom_prefix，而 buildSystemPrompt 把 custom_prefix 拼在 system 最前面——直接架空了同文件精心写的"角色锁定/防注入"最高优先级规则。

3. 游戏性维度

P0已落地

④ 缺"叙事预言机 / 混沌因子"——纯叙事没有真正的未知感

现状纯叙事模式所有情节由 LLM 采样决定，没有可控的随机性来源。玩家"不知道接下来发生什么"的惊喜感，完全依赖模型"吐"什么，不可设计、不可调难度。
同行单人跑团（Mythic GM Emulator）的灵魂就是 oracle：用"是/否/且/但"的随机判定 + 混沌因子 + 随机事件检定，让没有人类 GM 时依然有"命运的不确定性"。Ironsworn 的 move 也是掷骰决定成败与代价。
建议为纯叙事模式引入轻量 oracle：① 每 N 轮或小概率触发"随机事件/转折"（复用世界脉搏的 LLM 调用，但带" complication/ twist"意图）；② 提供一个"掷命运骰"按钮，玩家可对"某事是否发生"求 oracle，AI 据结果叙事；③ 维护一个"混沌值"，越高越易出意外。这让"单人跑团"的随机心跳成立。
成本中：需新增 oracle 状态 + UI 按钮 + prompt 段。
落地创建时可开关 oracle + 混沌因子滑杆(1-9)；每轮按 cf/9 概率注入随机事件；关闭则纯爽文；oracle_check 测试 8 PASS。
P1已落地

⑤ 世界是"文本"不是"模拟"：缺空间图 / 势力声望 / 经济 / 合成

现状地点只是 lore 里的名词；货币（铜钱）在背包里但无任何消耗处；无阵营/势力声望系统（只有全局 tension）；无装备/合成。NPC 有位置账本但无"世界一张网"。
同行AI Roguelite（Steam 在售，标榜"首个用 LLM 驱动真实游戏机制的 RPG"）把地点、NPC 日程、商店、物品、阵营全部做成可交互的模拟。玩家是在"玩游戏"，不只是"读故事"。
建议分阶段补"游戏骨架"：
  • 空间图：把 lore 地点连成有"旅行耗时/可达性"的图，移动消耗时间（与现有 periods 系统天然契合）；
  • 势力声望：把 tension 升级为多个阵营的 standing（友好/敌对影响可接任务与遭遇）；
  • 经济：让货币在商店/赏金/贿赂中真实流动；
  • 合成/装备：物品可组合、可穿戴改属性（跑团模式）。
成本高（这是"从引擎到游戏"的跃迁），但可先做空间图+势力声望两块，杠杆最大。
进度空间图地基（瓦片地图+玩家位置+战争迷雾）已落地，见 ⑭；本轮已补齐多阵营声望 factions / 货币经济 currency / 合成工坊 crafting / 装备系统——AI 在 state_changes 输出 factions/currency/crafting，玩家在「收集」面板装备/合成，武器伤害接入攻击检定。完整商店 NPC（固定货架/定价）与"阵营驱动任务/遭遇"留作增强。详见 docs/12 §10.5。

现状E12 传说摘要在新世界曾是"NPC 可能提起前世名字"的文本彩蛋；现已升级为 meta 成长：传说按声望/张力/达成目标/最深 NPC 羁绊/存活天数评为 传奇/史诗/显赫/平凡 四档，带入新世界时给予开局正反馈（初始声望/资源/特殊 NPC 关系/传奇档额外遗物），让重开有成长感（见 §10.9，legend_inherit_check 31 PASS）。

⑫ 成本 / 性能两处可优化

8. 总结

本项目已经是一个架构成熟度超出同体量开源项目的 AI 叙事引擎：记忆一致性、状态结构化、双模式、沉浸反馈闭环都做得很扎实，且在"不遗忘剧情"这一点上比开山鼻祖 AI Dungeon 还稳。

它已从"高质量的连续文本生成器"进化为带判定、带未知、带空间地图的 AI 单人跑团引擎（判定/预言机/常量记忆/地图战斗/章节幕/反向回写+战争迷雾均已落地）。对比 AI Roguelite 的"真游戏模拟"、NovelAI 的"可策展世界书"，本项目的下一步跃迁不在"文笔更好"，而在三件事：

判定已成立 ✅（技能检定） 未知已成立 ✅（oracle+混沌因子） 记忆已可控 ✅（pinned 常量层） 空间已成形 ✅（地图+战斗+迷雾） 世界已成形 ✅（势力/经济/合成/装备） IP 开箱即玩 ✅（预设库 8 世界+canon） 让记忆共编（玩家世界书）
